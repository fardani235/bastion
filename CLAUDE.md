# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Bastion is a local-first, cross-platform SSH connection manager with an encrypted credential vault. Wails v2 desktop app: Go backend + React/TypeScript/Vite/Tailwind frontend, xterm.js terminals. Secrets (passwords, key passphrases, AI API keys) live in a single SQLite DB encrypted under an Argon2id-derived master key.

## Commands

The `Makefile` wraps everything with the `webkit2_41` build tag, required on WebKit-4.1 systems (Ubuntu 24.04+, Fedora 38+):

```bash
make run        # hot-reloading dev app (wails dev)
make build      # production binary -> build/bin/
make deb        # build + package a Debian .deb
make clean
go test ./...                       # full Go test suite
go test ./internal/ssh/             # one package
go test ./internal/vault/ -run TestDerive   # one test
```

Frontend (rarely run directly — Wails drives it): `cd frontend && npm run build` runs `tsc && vite build`. There is **no frontend test runner and no Go linter configured** beyond `go vet` / `go test`.

Requirements: Go ≥ 1.25, Node ≥ 18, Wails CLI v2.12+. Linux needs GTK3 + WebKit2GTK dev packages.

## Architecture

**Three layers, deliberately decoupled.** The dependency rule is the key invariant:

- `internal/vault/` — crypto core (Argon2id KDF, AES-256-GCM, verify-blob). Knows nothing about storage; produces/consumes opaque ciphertext.
- `internal/store/` — SQLite persistence (hosts, groups, snippets, port_forwards, vault_meta). Stores ciphertext as opaque blobs; **never decrypts**.
- `internal/ssh/` — known-hosts trust, PTY sessions, port-forward manager, SFTP upload manager. **Vault-agnostic**: callers pass already-resolved plaintext credentials; output flows back through an `Emitter` interface, so the whole package is testable without Wails or a live SSH server.

`vault` and `store` do not import each other. Keep it that way — encryption boundaries depend on it.

**Root package (`main`) = the Wails IPC surface.** Each root `.go` file is one IPC domain, all methods hanging off the `App` struct (`app.go`). `App` owns the `*store.Store` and the in-memory 32-byte vault `key` (nil when locked), guarded by `App.mu`. The decryption boundary lives here: `hosts.go` decrypts credentials just-in-time before handing them to `internal/ssh`, and the renderer only ever receives `hasPassword`-style boolean flags — **plaintext secrets travel renderer→Go only, never Go→renderer.**

| File | IPC domain |
|---|---|
| `auth.go` | Setup / Unlock / Lock, KDF params |
| `hosts.go` | host CRUD + credential encrypt/decrypt boundary |
| `sessions.go` | OpenSession, host-key trust prompt flow, terminal I/O |
| `crud.go` | groups & snippets |
| `portforwards.go`, `hosts_import.go`, `transfer.go`, `ai.go` | port forwards / ~/.ssh/config import / SFTP upload / AI |
| `emitter.go` | Wails event emitter + optional session logging |

**Frontend.** `App.tsx` is a 4-state screen router (`loading → setup | unlock | main`) driven by vault state. State is a single Zustand store (`state/useAppStore.ts`) that mirrors server data (hosts/groups/snippets) plus client-only UI state (tabs, trust prompts, pending uploads). **All IPC goes through `lib/api.ts`** — a typed wrapper over the generated `wailsjs/go/main/App` bindings; components import from `lib/api.ts`, never the deep wailsjs paths. The wailsjs bindings are generated by `wails` from the Go `App` methods + DTO structs — regenerated on build, do not hand-edit.

**Events (Go→frontend) are per-session, name-suffixed by ID.** `runtime.EventsEmit` names: `session:output:<id>` (base64-encoded PTY bytes, decoded verbatim into xterm.js), `session:closed:<id>`, `upload:progress:<id>`, `upload:done:<id>`, and `vault:locked` (global — triggers auto-lock UI). Frontend subscribes via `EventsOn`.

## Conventions & invariants worth preserving

- **Adding an IPC method:** add the exported method to the right root `.go` file on `App`, then add a typed wrapper in `frontend/src/lib/api.ts`. Wails regenerates bindings on build.
- **Host-key trust is strict.** Unknown keys prompt once and pin to `known_hosts`; a *changed* key is a hard failure (possible MITM), never silently accepted. Don't loosen this.
- **Locking is destructive by design.** Lock/screensaver/shutdown closes every live SSH session, zeroes the in-memory key, and rejects further terminal input. A locked vault never leaves a writable terminal behind.
- **KDF params read back from `vault_meta` are bounds-checked** (`MinTime`/`MaxMemory` etc. in `vault.go`) to stop a tampered DB from forcing a weak or memory-exhausting KDF. Legacy PBKDF2-600k vaults still unlock.
- **Min master-password length (8) is enforced in the Go backend**, not just the UI.
- Port forwards bind `127.0.0.1` only. Uploads reuse the session's existing SSH connection (no re-auth) and read file bytes in Go — bytes never pass through the renderer.
- Remote CWD for upload destination is tracked from the shell's OSC 7 escape sequence (`internal/ssh/osc7_test.go`); falls back to home when absent.

## Platform notes

Drag-and-drop file upload is intentionally **not** used: on Linux/WebKit2GTK a dropped file navigates the webview to `file://` instead of yielding its path. Uploads go through a native file picker; `main.go` sets `DragAndDrop.DisableWebViewDrop` to neutralize stray drops. See the comment block in `main.go` before changing window options.

Vault location on Linux: `~/.config/bastion/vault.db` (dir created `0700` on first launch). Session logs (off by default): `~/.config/bastion/logs/`.
